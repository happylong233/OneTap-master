# OneTap 老人极简家人联系工具实现说明

## 结论

当前项目已经完成“老人极简家人联系工具”的核心改造，并且已经通过：

```powershell
.\gradlew.bat clean assembleDebug
```

构建产物：

```text
app/build/outputs/apk/debug/app-debug.apk
```

需要明确的是：普通电话、常用软件打开、老人桌面入口、家属设置中心这些能力可以通过系统 API 稳定实现。微信视频/语音电话依赖微信界面和无障碍节点，已经按安全策略实现，但必须在目标手机和目标微信版本上真机验证，不能承诺所有微信版本永久可用。

## 已实现功能

### 1. 老人首页

默认进入 `FamilyPage`，不是普通桌面，也不是通讯录列表。

已实现：

- 家人照片网格。
- 点击家人照片直接执行该联系人的默认联系方式。
- 联系人卡片提供两个大图标：
  - 摄像头：微信视频。
  - 电话听筒：普通电话。
- 页面底部提供“看视频”按钮进入常用软件页。
- 页面固定显示红色 SOS 紧急按钮。
- 右上角小齿轮：
  - 普通点击：语音提示“请让家人设置”。
  - 长按：进入家属设置中心。

### 2. 常用软件页

`AppsPage` 与 `FamilyPage` 分离，不把联系人和软件混在一起。

已实现：

- 只显示家属精选软件。
- 老人端最多显示 6 个软件。
- 点击大图标直接打开 App。
- 提供“找家人”按钮返回 `FamilyPage`。
- 页面固定显示红色 SOS 紧急按钮。
- 打不开软件时 TTS 提示：“这个软件没有打开成功，请让家人检查”。

### 3. 联系人模型

`Contact` 已支持老人模式需要的核心字段：

- `id`
- `name`
- `relationLabel`
- `phone`
- `wechatDisplayName`
- `wechatId`
- `avatarUri`
- `preferredCallMethod`
- `isFavorite`
- `isVisibleOnHome`
- `isEmergency`
- `order`

说明：

- 现有字段名 `name` 对应显示名称。
- 现有字段名 `phone` 对应手机号。
- 现有字段名 `order` 对应排序。
- `wechatDisplayName` 复用历史数据库列 `wechatNickname`，避免破坏已有数据。

### 4. 联系人管理

联系人管理只在家属设置中心中提供，不暴露在老人首页。

已实现：

- 添加联系人。
- 选择照片。
- 输入显示名称。
- 输入关系称呼。
- 输入手机号。
- 输入微信备注名。
- 输入微信号。
- 选择默认联系方式：
  - 微信视频。
  - 微信语音。
  - 电话。
- 设置是否显示在老人首页。
- 设置是否紧急联系人。
- 编辑或删除联系人。

未做的事：

- 没有预置任何默认联系人。
- 没有自动创建“女儿”“儿子”“孙子”“老伴”等联系人。
- 没有把示例联系人写入正式运行逻辑。

### 5. 微信视频 / 微信语音

已保留并重构微信无障碍方向。

流程：

1. 老人点击家人照片或摄像头按钮。
2. 如果默认方式是微信视频或微信语音，读取当前联系人的微信备注名或微信号。
3. 创建一次性微信自动化任务。
4. 打开微信。
5. 查找搜索入口。
6. 输入当前联系人的微信关键词。
7. 点击匹配该关键词的搜索结果。
8. 打开聊天页更多菜单。
9. 点击视频通话或语音通话。
10. 成功后停留在微信通话界面，等待接通。

语音提示：

- 启动视频：正在帮您打视频电话。
- 启动语音：正在帮您打开语音电话。
- 成功打开通话界面：已经帮您打开视频电话，请等待接通。
- 没有微信信息：这个家人还没有设置微信。
- 无障碍未开启：需要家人帮忙开启微信辅助。
- 已有任务执行中：正在帮您打开，请稍等。
- 失败：没有打开成功，请让家人帮忙检查微信。

重要限制：

- 不会说“已经接通”。
- 成功后不会自动按 Home。
- 不会把微信切到后台。
- 微信界面变化会影响成功率，需要真机验证。

### 6. 无障碍安全策略

已实现：

- 只处理 `com.tencent.mm`。
- 没有任务时不处理微信界面。
- 只有老人点击联系人照片、微信视频或微信语音后才创建任务。
- 已有任务时拒绝创建新任务。
- 任务成功、失败、超时后都会清理。
- 总超时约 15 秒。
- 没有 `Thread.sleep`。
- 没有无限循环。
- 没有固定坐标点击。
- 没有 `GLOBAL_ACTION_BACK` 强制乱退。
- 搜索结果必须匹配当前联系人的微信备注名或微信号。
- 找不到联系人或按钮时停止，不继续乱点。
- 不读取、不保存、不上传微信聊天内容。
- 不操作微信支付、红包、转账。
- 不把聊天内容写入日志。

硬编码微信控件 ID：

- 仍保留在 `WeChatId.kt`。
- 仅作为兜底。
- 主流程优先使用 text / contentDescription。
- 这些 ID 受微信版本影响，真机测试时需要重点关注。

### 7. 电话逻辑

普通联系人电话：

- 默认使用 `ACTION_DIAL` 打开拨号盘。
- TTS 提示：“已打开拨号页面，请点击绿色电话按钮”。
- 默认不直接拨出。

一键拨打电话：

- 只有家属在设置中心打开“一键拨打电话”后才使用 `ACTION_CALL`。
- 调用前由界面检查 `CALL_PHONE` 权限。
- 成功时 TTS 提示：“正在帮您拨打电话”。

紧急电话：

- SOS 支持 120、110 和紧急联系人。
- 使用 `ACTION_CALL`。
- 调用前检查电话权限。
- 不使用视频按钮组件。

### 8. 常用软件管理

家属设置中心中提供“常用软件管理”。

已实现：

- 扫描已安装 App。
- 搜索 App。
- 启用 / 停用精选软件。
- 最多选择 6 个。
- 支持上移 / 下移排序。
- 老人端只显示已启用软件。

未在老人端展示：

- 全部 App 抽屉。
- 应用分类。
- 系统应用管理。
- 复杂 App 列表。

### 9. 老人桌面模式

已保留 OneTap 作为默认桌面的能力。

已实现：

- `MainActivity` 同时保留 `HOME` 和 `LAUNCHER` 入口。
- 家属可以把 OneTap 设为默认桌面。
- 按 Home 回到 OneTap。
- `MainActivity.onResume()` 会重置到 `FamilyPage`。
- 不做设备所有者。
- 不做 Lock Task。
- 不禁用系统设置。
- 不隐藏原系统桌面。
- 不恶意拦截返回键。

退出老人桌面：

- 老人首页右上角齿轮长按进入家属设置中心。
- 家属设置中心提供“退出老人桌面”。
- 点击后跳转系统默认桌面设置页面。
- 家属在系统页面中选择原系统桌面。

### 10. 权限收敛

Manifest 已收敛到当前核心功能所需权限。

保留：

- `CALL_PHONE`
- `READ_EXTERNAL_STORAGE`
- `INTERNET`
- `ACCESS_NETWORK_STATE`
- `VIBRATE`
- `QUERY_ALL_PACKAGES`
- `com.android.permission.GET_INSTALLED_APPS`
- 微信无障碍服务声明

已移除：

- 联系人读取 / 写入权限。
- 相机权限。
- 悬浮窗权限。
- 开机自启权限。
- 位置权限。
- 精确闹钟权限。
- 通知权限。
- 写存储权限。

## 尚需真机验证的内容

以下功能编译层面已完成，但必须在目标设备验证：

1. 微信视频自动化。
2. 微信语音自动化。
3. 微信搜索结果是否能稳定匹配备注名或微信号。
4. 不同微信版本下“更多”“视频通话”“语音通话”节点是否能被识别。
5. 不同品牌系统的默认桌面设置页面是否能正确打开。
6. 常用软件扫描权限在 MIUI、ColorOS、OriginOS、HarmonyOS / EMUI 上的表现。
7. SOS 直拨权限授权流程。

## 真机测试步骤

### A. 基础安装

1. 安装 `app-debug.apk`。
2. 打开 OneTap。
3. 确认默认进入找家人页。
4. 确认首页没有天气、农历、新闻、广告、应用抽屉、系统管理入口。

### B. 添加联系人

1. 长按首页右上角齿轮 3 秒。
2. 进入家属设置中心。
3. 打开联系人管理。
4. 添加联系人：
   - 照片。
   - 显示名称。
   - 关系称呼。
   - 手机号。
   - 微信备注名。
   - 微信号。
   - 默认联系方式。
   - 是否显示在老人首页。
   - 是否紧急联系人。
5. 返回首页。
6. 确认联系人照片出现在 `FamilyPage`。

### C. 普通电话

1. 联系人默认方式设为电话。
2. 确认电话设置中“一键拨打电话”关闭。
3. 点击联系人照片。
4. 预期：打开拨号盘，不直接拨出。
5. 预期 TTS：已打开拨号页面，请点击绿色电话按钮。

### D. 一键电话

1. 家属设置中心打开“一键拨打电话”。
2. 授予电话权限。
3. 点击联系人电话图标。
4. 预期：直接拨出。
5. 预期 TTS：正在帮您拨打电话。

### E. 微信视频

1. 家属设置中心打开“微信一键辅助设置”。
2. 在系统无障碍中启用 OneTap 微信辅助。
3. 联系人填写正确的微信备注名或微信号。
4. 点击联系人照片或摄像头按钮。
5. 预期 TTS：正在帮您打视频电话。
6. 预期：微信被打开，搜索该联系人，进入聊天，打开视频通话。
7. 成功后预期 TTS：已经帮您打开视频电话，请等待接通。
8. 预期：不会自动返回 OneTap，不会说“已经接通”。

### F. 微信失败路径

1. 移除联系人的微信备注名和微信号。
2. 点击微信视频。
3. 预期 TTS：这个家人还没有设置微信。
4. 关闭无障碍服务。
5. 点击微信视频。
6. 预期 TTS：需要家人帮忙开启微信辅助。
7. 使用错误微信备注名。
8. 预期：15 秒左右失败并停止，不乱点。

### G. 常用软件

1. 家属设置中心打开常用软件管理。
2. 搜索并启用抖音、微信、相册、电话等。
3. 最多选择 6 个。
4. 调整顺序。
5. 返回老人页，点击“看视频”。
6. 确认只显示精选软件。
7. 点击软件图标。
8. 预期：直接打开软件。

### H. 老人桌面

1. 家属设置中心打开老人桌面模式。
2. 设置 OneTap 为默认桌面。
3. 按 Home。
4. 预期：回到 OneTap 的 `FamilyPage`。
5. 进入 AppsPage 后按 Home。
6. 预期：回到 `FamilyPage`，不是 AppsPage。

### I. 退出老人桌面

1. 长按右上角齿轮进入家属设置中心。
2. 点击“退出老人桌面”。
3. 预期：打开系统默认桌面设置页面。
4. 家属选择原系统桌面。
5. 按 Home。
6. 预期：回到原系统桌面。

## 当前风险

### 微信 UI 变化风险

微信自动化依赖微信无障碍节点。微信版本、字体大小、语言、A/B 实验、设备厂商定制都可能影响节点文本或层级。

缓解措施：

- 主流程优先查找 text / contentDescription。
- 搜索结果必须匹配联系人自己的关键词。
- 找不到节点就失败停止。
- 不使用固定坐标。
- 控件 ID 兜底集中在 `WeChatId.kt`。

### 权限和系统设置差异

默认桌面设置和应用列表权限在不同 Android 厂商系统上差异较大。

缓解措施：

- 退出老人桌面使用系统默认桌面设置入口。
- 常用软件管理保留应用列表权限引导。
- 真机测试必须覆盖最终给老人使用的手机型号。

## 是否已经“所有功能都可以实现”

从工程实现角度看，需求中的核心功能都已经有对应实现，并且项目可以 clean 构建。

从产品交付角度看，不能直接声称“所有功能在所有手机上都一定可用”。原因是：

- 微信视频/语音依赖第三方 App UI。
- 默认桌面设置依赖手机厂商系统。
- 应用列表权限依赖 Android 版本和厂商权限策略。

因此当前状态应定义为：

```text
核心功能已实现，构建通过；进入目标真机验收阶段。
```

真机验收通过后，才应定义为：

```text
可交付给老人使用。
```
